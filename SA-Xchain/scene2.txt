目前有橡胶工厂联盟、机械制造联盟和汽车制造商联盟三个组织，他们使用使用区块链来对自己的数据进行管理，我们后文称其为橡胶联盟链、机械联盟链和汽车联盟链，其中橡胶联盟链的链id为0x0000，机械联盟链的链id为0x0001，汽车联盟链的链id为0x0002。

各联盟及其链情况介绍：
橡胶工厂联盟会加工橡胶制品，在本案例中对汽车来说主要是轮胎，该工厂会生产两种不同规格的轮胎，分别是小型和大型，链中会存储小型和大型轮胎的当前库存，其中small_wheel_num代表小型轮胎数量，big_wheel_num代表大型轮胎数量。
橡胶联盟链当前数据如下：
{
    "small_wheel_num": "10",
    "big_wheel_num": "13"
}
机械制造联盟则会制造底盘及发动机等部件，该工厂会生产两种不同的底盘，分为高底盘和低底盘，链中会存储高底盘和低底盘的当前库存数量，其中high_chassis_num代表高底盘数量，low_chassis_num代表低底盘数量。
机械联盟链当前数据如下：
{
    "high_chassis_num": "2",
    "low_chassis_num": "3"
}
汽车制造商联盟会制造完整汽车，但是在本例中，汽车制造商只考虑汽车底座的制造，底座的定义是将轮胎和底盘组合之后的部件，汽车制造商会组合高低底盘和大小轮胎制作出四种不同的底座，A底座需要4个小轮胎和1个低底盘，B底座需要4个小轮胎和1个高底盘，C底座需要4个大轮胎和1个低底盘，D底座需要4个大轮胎和1个高底盘。在链上ABCD底座分别表示为base_a，base_b，base_c，base_d。
汽车联盟链当前数据如下：
{
    "base_a": "2",
    "base_b": "1",
    "base_c": "2",
    "base_d": "1"
}

制造协议：
每次汽车制造商有底座制造需求时，就会根据输入的制造的型号来查看目前各链的存货是否可以完成制造需求，如果不能完成就可以通过回滚退出交易，否则就减去对应链的部件的库存数量，增加汽车链的成品的数量。因此这里的输入参数只有base_type一个。

这里以输入参数base_type=base_a为例，已知A型号底座需要4个小轮胎和1个低底盘，首先从链id为0x0000的橡胶联盟链读取小轮胎small_wheel_num的数量为14，然后从链id为0x0001的机械联盟链读取低底盘low_chassis_num的数量为3，可以看到可以满足制造要求，那么将链id为0x0000的橡胶联盟链中小轮胎的库存修改为10，链id为0x0001的机械联盟链中低底盘的库存修改为2，然后给链id为0x0002的汽车联盟链增加一个A型底座，将目前A型底座的库存修改为3。

所以该操作结束之后，各链存储的数据将会变为如下：
橡胶联盟链当前数据如下：
{
    "small_wheel_num": "2",
    "big_wheel_num": "13"
}
机械联盟链当前数据如下：
{
    "high_chassis_num": "1",
    "low_chassis_num": "2"
}
汽车联盟链当前数据如下：
{
    "base_a": "3",
    "base_b": "2",
    "base_c": "2",
    "base_d": "1"
}

由于本场景各测试用例之间是有关系的，比如本次制造完成之后低底盘只剩一个或者零个，下一次将无法再使用低底盘制造底座。而输出只会输出数据不正确的用例，所以这里给出所有用例输入。
1.base_a
2.base_b
3.base_c
4.base_d
5.base_d
6.base_a
